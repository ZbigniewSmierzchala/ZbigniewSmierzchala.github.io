01_CASE STUDY: Event_Planner

Transitioning from rigid Excel sheets to a tailor-made web application.

The Challenge: Standard software was either too complex or too inflexible for the specific administrative processes of a German local authority. The goal was to build a tool that adapts to the user—not the other way around.
  • User-Centric: Focused on the 20% of features that deliver 80% of the value.
  • Data Sovereignty: 100% local storage to comply with strict privacy regulations.
  • Sustainability: Capturing institutional knowledge for future event cycles.
HTML5 | CSS3 | JavaScript | PHP | Docker | Ubuntu Server

02_SYSTEM_ARCHITECTURE: Local-First Design

A lean, secure, and maintenance-free stack built for reliability.

  • Frontend Stack: Vanilla JavaScript (ES6+), HTML5 & CSS3. No external libraries, ensuring high performance and zero-dependency maintenance.
  • Data Security: Persistence via Browser LocalStorage, featuring an intelligent change-tracking system and safety prompts.
  • Infrastructure: Optimized as a portable web app, ready for deployment via Docker & Portainer on Ubuntu servers.

03_DEPLOYMENT_IMPACT

Boosting efficiency through intelligent user guidance.

Designed to reduce human error and keep process ownership within the team.

  • Local & Secure: Privacy by design—no data ever leaves the local environment.
  • Deadline Alerts (Fristen-Warnung): Built-in visual cues for overdue tasks and milestones.
  • Experience Templates: Leverage successful events as "Blueprints" for the following year to prevent data bloat.
  • Error Tolerance: Safety confirmations to prevent the accidental deletion of critical project phases.
